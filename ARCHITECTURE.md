## Компоненты

### 1. Фронтенд (статический сайт)
- **Технологии**: HTML5, CSS3, Vanilla JavaScript
- **Стили**: Фирменный стиль 2ГИС ФИТ (оранжевый #FF5722, чёрный, белый)
- **Функции**:
  - Отображение видеопримеров танцевальных направлений
  - Форма сбора данных
  - Валидация ввода
  - Авто-сохранение (localStorage)
  - Отправка данных через fetch API

### 2. n8n (workflow automation)
- **Технология**: n8n (self-hosted)
- **Узлы**:
  - **Webhook**: приём POST запросов с формы
  - **Google Sheets**: сохранение данных в таблицу
  - **Telegram**: отправка уведомлений
  - **Respond**: ответ клиенту
- **Триггер**: POST запрос на webhook URL

### 3. Хранение данных
- **Google Sheets**: основная таблица с ответами
- **Telegram**: уведомления в реальном времени
- **SQLite/PostgreSQL**: опционально для истории выполнений n8n

## Развёртывание

### Вариант 1: Локальный (для тестирования)
```
Фронтенд: http://localhost:8000
n8n:      http://localhost:5678
```

### Вариант 2: Docker (рекомендуется)
```yaml
services:
  n8n:    port 5678
  nginx:  port 80 (фронтенд + proxy webhook)
```

### Вариант 3: IIS + n8n
```
Фронтенд: IIS на порту 80
n8n:      отдельный процесс на порту 5678
```

### Вариант 4: Nginx + n8n (Linux)
```
Фронтенд: Nginx на порту 80
n8n:      порт 5678 (proxy через nginx)
```

## Безопасность

### Внутренний контур
- Все сервисы внутри корпоративной сети
- Нет доступа из интернета
- Firewall правила ограничивают доступ

### Рекомендации
1. **HTTPS**: используйте SSL сертификаты
2. **Аутентификация**: добавьте login/password для n8n UI
3. **CORS**: настройте разрешённые домены
4. **Валидация**: проверяйте данные на входе
5. **Бэкапы**: регулярно сохраняйте Google Sheets

## Поток данных

1. Сотрудник открывает сайт → загружается статика
2. Сотрудник заполняет форму → данные валидируются
3. Форма отправляется → POST на webhook n8n
4. n8n получает данные → workflow запускается
5. Данные сохраняются → Google Sheets
6. Уведомление отправляется → Telegram
7. Ответ возвращается → браузер показывает успех

## Масштабирование

### Если много ответов:
- Подключите PostgreSQL вместо SQLite
- Добавьте очередь (Redis) для обработки
- Настройте rate limiting в n8n

### Если нужен доступ извне:
- Настройте VPN
- Используйте reverse proxy с SSL
- Добавьте аутентификацию

## Мониторинг

### Проверка работы:
- n8n executions: `http://server:5678/executions`
- Логи nginx: `/var/log/nginx/dance-survey.*.log`
- Google Sheets: проверяйте новые строки
- Telegram: проверяйте сообщения

### Метрики:
- Количество ответов в день
- Конверсия (посетители → ответы)
- Время обработки (webhook → сохранение)
- Ошибки (5xx в логах)

## Обновления

### Фронтенд:
1. Обновите файлы в `/var/www/dance-survey`
2. Перезагрузите nginx (если нужно)

### n8n workflow:
1. Экспортируйте текущий workflow
2. Внесите изменения
3. Импортируйте новый
4. протестируйте на тестовых данных

### Docker:
```bash
docker-compose pull
docker-compose up -d
```
